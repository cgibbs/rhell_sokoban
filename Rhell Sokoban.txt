Need to be able to process instantaneous movement versus turn-based movement, e.g. "moving across a series of arrow tiles" or "arrow tile moves player into pushing a block" as opposed to "player has moved and now we need to update the board". Treating them the same will create situations where the player gets moved by tiles and causes a lot of board updates, which can cause stateful blocks with timer variables (e.g. water turns to steam if left by fire too long) to pop their effects before the player ever gets a chance to react. It also prevents situations like "a fire tile is moving past several ice tiles via arrow tiles and melts all of them", because the melt effect would only happen on a full board update.
